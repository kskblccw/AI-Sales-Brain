# 电商客服系统 -- 多模态功能方案

## 一、多模态 = 什么

当前系统只处理文本。多模态扩展后：

| 模态 | 用户能做什么 | 系统需要什么 |
|------|-------------|-------------|
| 图片 | 拍破损商品/截图订单/拍尺码标签 | 视觉 LLM 理解图片内容 |
| 语音 | 说话代替打字 | ASR 语音转文字 |
| 视频 | 展示商品使用问题 | 关键帧提取 + 视觉理解 |

## 二、架构设计

```
用户输入（文本 / 图片 / 语音）
  │
  ├── 文本 → 直接进入现有 pipeline
  │
  ├── 图片 → POST /api/upload → 存储到 local/OSS
  │              │
  │              ├── 视觉 LLM 理解 → "用户上传了一张订单截图，订单号 ORDxxx"
  │              └── 文本描述注入消息 → 现有 Agent pipeline
  │
  └── 语音 → 前端 Web Audio API 录制
                │
                ├── POST /api/speech-to-text → ASR → 文本
                └── 文本注入消息 → 现有 Agent pipeline
```

核心原则：**多模态只在入口做一次转换，转换成文本后复用现有的全部 Agent 逻辑**。

## 三、图片理解（最高优先级）

### 3.1 为什么图片能力最重要

电商客服场景中图片是刚需：

- 用户拍破损商品 → 需要识别损坏程度
- 用户截订单图 → 需要 OCR 提取订单号（比手打更准确）
- 用户拍尺码标签 → 需要判断尺码是否合适
- 用户拍商品外观 → 需要判断是否为假货

### 3.2 实现方案

```
POST /api/upload/{session_id}
  Content-Type: multipart/form-data
  Body: image + optional_description

处理流程:
  1. 保存图片到 uploads/{session_id}/{timestamp}.jpg
  2. 调用视觉 LLM（qwen-vl-plus）分析图片
     prompt: "这是一张电商客服场景的图片。请描述图片内容，
              如果是订单截图请提取订单号，如果是商品请描述状态。"
  3. 返回分析文本
  4. 将 "用户上传了一张图片：[分析文本]" 作为 HumanMessage 注入
  5. 走现有的意图识别 → Agent 路由流程
```

### 3.3 代码骨架

```python
# server.py 新增端点
from fastapi import UploadFile, File

@app.post("/api/upload/{session_id}")
async def upload_image(session_id: str, file: UploadFile = File(...)):
    # 1. 保存
    upload_dir = Path("uploads") / session_id
    upload_dir.mkdir(parents=True, exist_ok=True)
    path = upload_dir / f"{int(time.time())}_{file.filename}"
    content = await file.read()
    path.write_bytes(content)

    # 2. 视觉 LLM 分析
    import base64
    img_b64 = base64.b64encode(content).decode()

    from langchain_core.messages import HumanMessage
    vision_llm = ChatOpenAI(
        model="qwen-vl-plus",        # 阿里云多模态模型
        base_url=DASHSCOPE_BASE_URL,
        api_key=DASHSCOPE_API_KEY,
    )
    response = vision_llm.invoke([
        HumanMessage(content=[
            {"type": "text", "text": "请描述这张图片。如果是订单截图提取订单号，"
             "如果是商品请描述外观和可能的问题。简短回复。"},
            {"type": "image_url", "image_url": {"url": f"data:image/jpeg;base64,{img_b64}"}},
        ])
    ])

    return JSONResponse({
        "analysis": response.content,
        "image_url": f"/uploads/{session_id}/{path.name}",
    })
```

### 3.4 前端改动

```javascript
// app.js 新增图片上传
function handleImageUpload(file) {
    const formData = new FormData();
    formData.append("file", file);

    // 先显示图片缩略图
    addMessage("user", `<img src="${URL.createObjectURL(file)}" class="chat-image">`);

    fetch(`/api/upload/${getCurrentSessionId()}`, {
        method: "POST",
        body: formData,
    })
    .then(r => r.json())
    .then(data => {
        // 将分析文本当作用户消息发送
        chatInput.value = `[用户上传图片] ${data.analysis}`;
        sendMessage();
    });
}

// 拖拽或粘贴图片触发上传
chatInput.addEventListener("paste", (e) => {
    const file = e.clipboardData.files[0];
    if (file && file.type.startsWith("image/")) {
        handleImageUpload(file);
    }
});
```

## 四、语音输入

### 4.1 方案

前端录制 → 后端 ASR → 文本注入消息流。

```
前端: MediaRecorder API 录制 → WAV
  → POST /api/speech-to-text
  → 后端: 调用阿里云 ASR（Paraformer）→ 文本
  → 返回文本 → 前端填入输入框或自动发送
```

### 4.2 后端端点

```python
@app.post("/api/speech-to-text")
async def speech_to_text(file: UploadFile = File(...)):
    import dashscope
    audio_bytes = await file.read()
    audio_b64 = base64.b64encode(audio_bytes).decode()

    # DashScope 语音识别
    response = dashscope.audio.asr.Transcription.call(
        model="paraformer-v2",
        file_urls=[f"data:audio/wav;base64,{audio_b64}"],
    )
    text = response.output.results[0].transcription

    return JSONResponse({"text": text})
```

### 4.3 前端录制

```javascript
let mediaRecorder, audioChunks = [];

async function startRecording() {
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    mediaRecorder = new MediaRecorder(stream);
    audioChunks = [];
    mediaRecorder.ondataavailable = e => audioChunks.push(e.data);
    mediaRecorder.onstop = async () => {
        const blob = new Blob(audioChunks, { type: "audio/wav" });
        const form = new FormData();
        form.append("file", blob);
        const r = await fetch("/api/speech-to-text", { method: "POST", body: form });
        const d = await r.json();
        chatInput.value = d.text;
        sendMessage();
    };
    mediaRecorder.start();
    // UI: 显示录音中状态...
}

function stopRecording() {
    mediaRecorder.stop();
}
```

## 五、实施优先级

| 优先级 | 功能 | 工作量 | 价值 |
|--------|------|--------|------|
| P0 | 图片上传 + 视觉 LLM 理解 | 2-3 天 | 极高：解决退货/投诉中的图片证据场景 |
| P1 | 粘贴/拖拽图片 | 0.5 天 | 高：用户体验 |
| P2 | 语音输入 | 1-2 天 | 中：移动端场景 |
| P3 | 视频关键帧 | 3-5 天 | 低：低频场景 |

## 六、多模态对现有 Agent 的影响

最小侵入原则：多模态只在消息入口做转换。

```
之前：用户输入文本 → HumanMessage → Agent
之后：用户输入图片 → 视觉 LLM → 文本描述 → HumanMessage → Agent
     用户输入语音 → ASR → 文本 → HumanMessage → Agent
```

所有 Agent、工具、路由逻辑完全不变。唯一的改动是入口处多了一个"翻译层"。

## 七、多模态下的安全考虑

1. **图片内容审核**：用户上传的图片可能包含敏感内容，建议接入阿里云内容安全 API 做前置审核
2. **图片隐私**：订单截图可能包含他人信息（收件人姓名/地址/电话），视觉 LLM 分析时应脱敏后再注入消息
3. **语音防注入**：ASR 转写后的文本应经过与文本输入相同的过滤逻辑
4. **存储管理**：上传的图片/音频应有自动过期清理策略（如 14 天）
