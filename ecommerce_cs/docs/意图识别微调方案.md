# 意图识别模型微调方案

## 一、为什么要微调

当前意图识别依赖两层：正则预路由（80% 命中）→ LLM 分类器（qwen-plus，兜底 20%）。

LLM 分类器的问题：

| 问题 | 表现 |
|------|------|
| 延迟 | 每次 LLM 调用 0.5-2 秒，用户体验差 |
| 成本 | 每个未命中正则的请求都消耗 token |
| 不稳定 | 同一个问题可能被分到不同意图（受记忆上下文干扰） |
| 寒暄误判 | "你好""在吗"可能被分到 aftersale |

**目标**：用微调后的小模型（0.5B-1.5B 参数）替代 LLM 分类器，本地推理，<50ms 延迟，零 token 成本。

## 二、整体架构

```
用户消息
  │
  ├── 第1层: pre_router_node（正则盲筛，不变）
  │     命中 → 直接返回 intent
  │     未命中 → intent="__llm__"
  │
  └── 第2层: intent_classifier_node
        之前: LLM (qwen-plus) 远程调用
        之后: 微调模型 本地推理          ← 只改这里
              │
              ├── 加载本地 ONNX/GGUF 模型
              ├── 推理（<50ms）
              └── 返回 intent 标签
```

只替换 `intent_classifier_node` 中的 LLM 调用，其他部分完全不变。

## 三、数据准备

### 3.1 数据来源

| 来源 | 数量（估计） | 质量 |
|------|-------------|------|
| LangSmith 评估日志 | 30 条已标注 | 高 |
| 正则命中日志（服务端 print） | 可从日志提取 | 高（正则结果即标签） |
| 人工构造 | 可扩 200+ 条 | 可控 |
| LLM 分类日志（作为弱标注） | 每次兜底调用 | 中（需人工抽检） |

### 3.2 数据格式

每条训练样本为 JSONL 一行：

```json
{
  "text": "帮我查一下我的订单",
  "label": "order"
}
{
  "text": "你们支持花呗分期吗",
  "label": "faq"
}
{
  "text": "我买的耳机有杂音怎么退货",
  "label": "aftersale"
}
{
  "text": "推荐一款扫地机器人",
  "label": "product"
}
{
  "text": "你好",
  "label": "faq"
}
{
  "text": "在吗？",
  "label": "faq"
}
{
  "text": "转人工",
  "label": "human"
}
```

### 3.3 标签分布要求

5 个类别（order / product / aftersale / faq / human），每类至少 80 条，总计 400+ 条。重点关注容易混淆的边界样本：

- "下单后几天能收到货" → faq（不是 order）
- "谢谢你的帮助" → faq（不是 aftersale）
- "我要退货退款" → aftersale（不是 order）
- "上周买的还没到" → order（不是 aftersale）

### 3.4 数据增强策略

通过同义替换生成更多样本：

```
原始: "帮我查一下我的订单"
增强: "看看我的订单" / "我的订单有哪些" / "查订单" / "订单列表"
```

每类至少扩到 150 条，总计 750+ 条。

## 四、模型选型

### 4.1 候选模型

| 模型 | 参数量 | 推理速度(CPU) | 优势 |
|------|--------|--------------|------|
| Qwen2.5-0.5B-Instruct | 0.5B | <20ms | 最小，部署最简单 |
| Qwen2.5-1.5B-Instruct | 1.5B | <50ms | 平衡速度和效果 |
| ModernBERT-base | 139M | <10ms | 纯编码器，专为分类设计 |
| BGE-M3 微调为分类 | 568M | <15ms | 已有中文预训练 |

### 4.2 推荐方案：ModernBERT-base

```
选型理由：
  - 纯 Encoder 架构，天然适合文本分类（不需要生成能力）
  - 139M 参数，CPU 推理 <10ms，可部署在任何机器
  - HuggingFace Trainer 一行代码微调
  - 支持中文（multilingual 版本）
```

备选：如果效果不达预期，升级到 Qwen2.5-0.5B-Instruct（Decoder 架构但更强大）。

### 5.2 训练脚本

```python
# train_intent_classifier.py
from transformers import (
    AutoTokenizer, AutoModelForSequenceClassification,
    Trainer, TrainingArguments, DataCollatorWithPadding
)
from datasets import Dataset
import json
import numpy as np
from sklearn.metrics import accuracy_score, f1_score

# 1. 加载数据
with open("intent_data.jsonl") as f:
    data = [json.loads(line) for line in f]

label2id = {"order": 0, "product": 1, "aftersale": 2, "faq": 3, "human": 4}
id2label = {v: k for k, v in label2id.items()}

texts = [d["text"] for d in data]
labels = [label2id[d["label"]] for d in data]

dataset = Dataset.from_dict({"text": texts, "label": labels})
dataset = dataset.train_test_split(test_size=0.15, seed=42)

# 2. 加载模型
model_name = "answerdotai/ModernBERT-base"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(
    model_name, num_labels=5, id2label=id2label, label2id=label2id
)

def preprocess(examples):
    return tokenizer(examples["text"], truncation=True, max_length=64)

tokenized = dataset.map(preprocess, batched=True)

# 3. 训练
def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = np.argmax(logits, axis=-1)
    return {"accuracy": accuracy_score(labels, preds), "f1": f1_score(labels, preds, average="macro")}

trainer = Trainer(
    model=model,
    args=TrainingArguments(
        output_dir="./intent_model",
        num_train_epochs=5,
        per_device_train_batch_size=32,
        per_device_eval_batch_size=32,
        learning_rate=2e-5,
        warmup_ratio=0.1,
        weight_decay=0.01,
        logging_steps=10,
        eval_strategy="epoch",
        save_strategy="epoch",
        load_best_model_at_end=True,
    ),
    train_dataset=tokenized["train"],
    eval_dataset=tokenized["test"],
    data_collator=DataCollatorWithPadding(tokenizer),
    compute_metrics=compute_metrics,
)

trainer.train()

# 4. 导出
trainer.save_model("./intent_model/final")
tokenizer.save_pretrained("./intent_model/final")
```

### 5.3 目标指标

| 指标 | 目标 |
|------|------|
| Accuracy | >92% |
| Macro F1 | >0.90 |
| 推理延迟 | <20ms (CPU) |
| 模型大小 | <500MB 磁盘 |

## 六、集成到现有系统

### 6.1 加载模型

```python
# config.py 新增
from transformers import AutoTokenizer, AutoModelForSequenceClassification
import torch

_intent_model = None
_intent_tokenizer = None
_intent_labels = ["order", "product", "aftersale", "faq", "human"]

def get_intent_classifier():
    global _intent_model, _intent_tokenizer
    if _intent_model is None:
        model_path = Path(__file__).parent / "intent_model" / "final"
        _intent_tokenizer = AutoTokenizer.from_pretrained(str(model_path))
        _intent_model = AutoModelForSequenceClassification.from_pretrained(str(model_path))
        _intent_model.eval()
    return _intent_model, _intent_tokenizer

def classify_intent(text: str) -> str:
    """本地推理，返回意图标签"""
    model, tokenizer = get_intent_classifier()
    inputs = tokenizer(text, return_tensors="pt", truncation=True, max_length=64)
    with torch.no_grad():
        logits = model(**inputs).logits
    idx = torch.argmax(logits, dim=-1).item()
    return _intent_labels[idx]
```

### 6.3 灰度切换

```
初期：微调模型 + LLM 双路并行
  ├── 微调模型作为主路径 → 返回 intent
  └── LLM 作为影子路径 → 仅日志记录，对比差异

稳定期（连续 1000 条一致率 >95%）：
  └── 微调模型单独使用，LLM 路径移除
```

## 七、持续优化

### 7.1 在线学习

```
intent_classifier_node 推理时：
  ├── 模型给出 intent + confidence
  ├── confidence < 0.7 → 标记为"低置信度"
  └── 低置信度样本自动写入 pending_review.jsonl
```

人工定期审核低置信度样本 → 纠正标签 → 加入训练集 → 每月重新微调。

### 7.2 评估闭环

```
LangSmith 评估（30 条）每次微调后跑一次
  ├── 意图准确率 vs 之前的 LLM 版本
  ├── 每类 Precision/Recall
  └── 记录 confidence 分布
```

### 7.3 模型蒸馏（进阶）

如果 ModernBERT-base 效果不够，可以用 LLM（qwen-plus）生成 5000 条标注数据，蒸馏到小模型。LLM 的标注质量通常很高（电商客服场景），蒸馏后的小模型可接近 LLM 的准确率。
